マーケ・SNS担当のためのChatGPTプロンプト10選

そのままコピペで使える、業務特化テンプレ集です。

はじめに

マーケティングやSNS運用の現場では、「考える前の整理」「最初のたたき」「ちょっとした言い換え」に時間が取られがちです。

このテンプレ集は、そういう「最初の一手間」をChatGPTに任せて、人間は判断と仕上げに集中できるようにするためのものです。

使い方はシンプルで、該当するテンプレをそのままChatGPTに貼り付けて、{ }で囲まれた変数だけを自分の状況に置き換えるだけです。

10種類すべて、初心者の方でも今日から使える内容にしています。

ChatGPTを主軸に書いていますが、ClaudeやGeminiでも同じように動くものが多いです。

1. SNS投稿の原案たたき

用途

SNS投稿の最初のたたきを作る。ゼロから書き出すのが重い時に、構成と切り口の候補を一気に出してもらう。

そのまま貼れるプロンプト

あなたはSNS運用の経験が長いライターです。
以下の条件で、{プラットフォーム}向けの投稿原案を3案出してください。

- テーマ: {投稿テーマ}
- 想定読者: {読者像}
- 投稿の目的: {目的（例: 保存される / コメントが来る / 共感される）}
- 文字数の目安: {文字数}
- 避けたい表現: 煽り、誇張、断定

各案について、以下を必ず含めてください。
1. 冒頭1〜2行（フック）
2. 本文（指定文字数内）
3. この案の狙い（なぜこの切り口なのか）
4. 注意点（この案の弱点・誤解されそうな部分）

入れ替える変数

  • {プラットフォーム}: Threads / X / Instagram / LinkedIn など
  • {投稿テーマ}: 具体的なテーマを1〜2文で
  • {読者像}: 年代・職種・関心
  • {目的}: 1つに絞る
  • {文字数}: 例「250〜350字」

使うときの注意

  • そのまま投稿せず、たたき台として扱う。違和感のある言い回しは必ず手直しする
  • 同じテーマで何度か出すと、似た構成に偏ることがある。切り口を変えたい時は「全く別の角度で」と追加で指示する

出力を良くする追加指示

  • 「投稿の冒頭で『〜ですよね』のような共感型は使わないでください」など、避けたい型を具体的に書く
  • 過去に伸びた自社投稿を1〜2本貼って「このトーンに寄せて」と渡すと、トーンのブレが減る

2. 見出し量産

用途

ブログ・noteの見出し、LP内の小見出し、メルマガのテーマ案など、見出し系を一気に量産する。

そのまま貼れるプロンプト

以下の本文または企画概要を読んで、見出し候補を10個出してください。

- 用途: {用途（ブログタイトル / 章見出し / LP見出し など）}
- 本文または概要:
{本文 or 企画概要}

条件:
- 文字数は{目安文字数}以内
- 数字や固有名詞を入れたバージョンも混ぜる
- 読者が「自分のことだ」と感じる切り口を1つは入れる
- 煽り・誇張は避ける（「絶対」「完全」「最強」などはNG）
- 10個の中で重複しないように、切り口を分散させる

各見出しの下に、なぜその切り口にしたかを1行で添えてください。

入れ替える変数

  • {用途}: 何の見出しなのか
  • {本文 or 企画概要}: 元になる素材
  • {目安文字数}: 例「30字」「20字」

使うときの注意

  • 出てきた見出しをそのまま使うと、表現が均質になる傾向がある。最終的に2〜3案に絞り、人間が言葉選びを微調整する
  • AIは「数字を入れすぎる」「断定しすぎる」傾向があるので、最終チェックで弱める

出力を良くする追加指示

  • 「自社のブランドトーンに合わせて、丁寧語・敬体で書いてください」と一文足すと、トーンが整う
  • 「読者の悩み起点」「ベネフィット起点」「数字起点」「対比起点」のように切り口の枠を指定すると、バリエーションが揃う

3. ペルソナ抽出

用途

お客様の声・問い合わせ・アンケート回答などのまとまった情報から、ペルソナの輪郭を引き出す。

そのまま貼れるプロンプト

以下の素材から、サービスの想定顧客像（ペルソナ）を抽出してください。

素材:
{お客様の声 / アンケート結果 / 問い合わせログ などをそのまま貼る}

出力形式:
1. 共通点が多い読者像 上位3パターン
2. 各パターンについて以下を埋める
   - 年代・職業・立場の推定
   - 抱えていそうな課題（素材から読み取れるもの）
   - 解決したいゴール
   - 使っている言葉のクセ（素材から拾える表現）
   - 響きそうな訴求の方向性
3. 注意点として、素材から読み取れない部分（推測になっている部分）は明示してください

入れ替える変数

  • {素材}: 実際の声・回答・ログ。匿名化してから貼ること

使うときの注意

  • 個人情報・社内秘情報は必ず匿名化してから貼る
  • AIは「ありそうなペルソナ」を作る傾向があるので、素材から読み取れない部分は推測だと明示させる
  • 出てきたペルソナはそのまま採用せず、社内メンバーで「実在のお客様像と合うか」を必ず照合する

出力を良くする追加指示

  • 「年代や職業を具体的に断定せず、『〜の傾向がある』と幅を残してください」と指示すると、現実離れしにくい
  • 素材が少ない時は、「データが薄いので、断定せず仮説として書いてください」と最初に伝える

4. 競合分析の最初の整理

用途

競合のサイト・SNS・サービス紹介ページなどを並べた時の、最初の整理。意思決定のたたき台に使う。

そのまま貼れるプロンプト

以下の競合情報を整理してください。

自社の概要:
{自社サービスの一言説明}

競合情報:
- 競合A: {サイト要約 or 訴求文}
- 競合B: {サイト要約 or 訴求文}
- 競合C: {サイト要約 or 訴求文}

出力してほしいこと:
1. 各競合の主訴求（一言）
2. 各競合の強そうな点・弱そうな点
3. 自社と被っている訴求 / 自社と差別化できそうな空白
4. 「ここを攻めると埋もれる」リスクのあるポジション
5. 推測している部分は最後にまとめて明示

トーン: 断定しすぎず、「〜の可能性がある」「〜と読み取れる」のように整理してください。

入れ替える変数

  • {自社サービスの一言説明}: 短く
  • {競合A〜Cのサイト要約}: 各社の公開情報を貼る。要約済みでもOK

使うときの注意

  • これはあくまで「最初の整理」。最終的な競合分析は人間の現場感で判断する
  • AIは公開情報から書ける範囲しか見ていない。価格戦略・営業現場の動きなどは出ないので過信しない
  • 競合の悪口や格下げ表現が混ざることがあるので、出力後に必ず確認する

出力を良くする追加指示

  • 「競合の評価で否定的な強い表現（時代遅れ、ダメ、など）は使わないでください」と最初に伝える
  • 「自社の強みは何かを提示するのではなく、空白ポジションの候補を出してください」と限定すると、情報商材的な答えになりにくい

5. 反応が薄かった理由の仮説出し

用途

投稿・メルマガ・LPなど、思ったより反応が薄かった施策について、原因仮説を複数出す。

そのまま貼れるプロンプト

以下の施策が、想定より反応が薄かった理由の仮説を5つ出してください。

施策の内容:
{投稿本文 or メルマガ本文 or LPの抜粋}

数値:
- 想定: {想定していた反応の例}
- 実際: {実際の反応の例}

文脈:
- 配信先: {配信先 / 投稿先}
- 配信時期や状況で気になる点: {あれば記入}

出力形式:
仮説1〜5について、以下を埋める
- 仮説の内容
- そう考える根拠（本文・数値・文脈のどこから読み取れるか）
- この仮説を確かめるには次に何を検証すればよいか
- 仮説の確度（高 / 中 / 低）とその理由

注意:
- 「読者の理解度が低かった」のような相手のせいにする仮説は、根拠がある時だけ含める
- 5つの仮説は、できるだけ別の角度から出してください（コピー、構成、タイミング、媒体、訴求対象 など）

入れ替える変数

  • {施策の内容}: 該当する施策本文
  • {想定 / 実際}: 簡単な数値や反応で良い
  • {配信先 / 投稿先}: プラットフォームや読者層

使うときの注意

  • 「読者が悪い」系の仮説を強く出す傾向があるので、根拠があるものに限定するよう指示する
  • 確度の自己評価が甘くなりがち。出てきたものは社内で再評価する
  • 1施策の反応だけで結論を出さない。仮説止まりにする

出力を良くする追加指示

  • 「施策側で改善できるポイントに絞って5つ出してください」と限定すると、行動につながる仮説になる
  • 過去の似た施策のデータがあれば貼ると、仮説の精度が上がる

6. メール件名AB案

用途

配信メール・営業メール・問い合わせメールなどの件名を、AB案として複数比較する。

そのまま貼れるプロンプト

以下のメールの件名候補を、A群とB群に分けて出してください。

メール本文の概要:
{本文 or 概要}

配信先:
{配信先の属性}

メールの目的:
{目的（開封してもらう / 返信してもらう / 行動してもらう など）}

A群の方針: ベネフィット先出し型（読者の利益を冒頭で示す）
B群の方針: 課題提起型（読者の困りごとを先に置く）

各群について
- 件名候補を5つずつ
- 各件名の文字数（全角換算）
- 各件名の狙い（1行）
- 注意点（誤解されそう / 強すぎる / 弱すぎる など）

避けたい表現:
- 「絶対」「必ず」「最後の」「特別」「限定」などの煽り
- 開いたら本文と内容が違うクリックベイト

入れ替える変数

  • {本文 or 概要}: メール内容
  • {配信先の属性}: 既存顧客 / 見込み客 / 問い合わせ後 など
  • {目的}: 1つに絞る

使うときの注意

  • 件名は最終的にメーラーやスマホ画面で見え方が変わる。文字数だけでなく実機で確認する
  • 「【】」「!」などの記号は、配信先の業界・関係性によって受け取られ方が違う。まずは記号控えめでテストする
  • AB案は同時配信できる環境がない場合、週ずらしで検証する

出力を良くする追加指示

  • 「文字数は28字以内でお願いします（モバイル表示で切れない範囲）」と具体的に指定する
  • 配信ツールの仕様（プリヘッダーが見えるか）も伝えると、件名とプリヘッダーをセットで提案してくれる

7. CTAリライト

用途

LP・投稿・メルマガ・LINEなどの行動喚起文（CTA）を、強さ違い・トーン違いで複数案にする。

そのまま貼れるプロンプト

以下のCTA文を、3パターンにリライトしてください。

元のCTA:
{現状のCTA}

文脈:
- 置かれている場所: {例: LPの最後 / 投稿末尾 / メールの結び}
- 読者の状態: {例: まだサービスを知ったばかり / 何度か接点あり / 既存顧客}
- 行動の重さ: {例: 無料登録 / 資料請求 / 問い合わせ / 購入}

3パターンの方針:
- パターン1: 行動の重さを軽く感じさせるトーン
- パターン2: 行動の理由（ベネフィット）を明確にするトーン
- パターン3: 押し売り感ゼロで「保留してもよい」雰囲気のトーン

各パターンに以下を添えてください。
- リライト案
- 元のCTAとの違いの一言要約
- このパターンが向く読者の状態

避けたい表現: 「今すぐ」「絶対」「限定」「逃すと損」のような煽り。

入れ替える変数

  • {現状のCTA}: 今使っているCTA文をそのまま貼る
  • {文脈}: 場所・読者・行動の重さ

使うときの注意

  • CTA1つだけ強くしても、本文が弱いと反応は変わらない。本文との温度差をチェックする
  • リライト案は「自社らしさ」が薄まりがち。最後はブランドのトーンに揃える
  • 同じLPで毎月リライトしすぎると、検証データが取れなくなる。検証期間を決めてから差し替える

出力を良くする追加指示

  • 「読者が断りづらく感じる表現は避けてください」と入れると、心理的圧の強い案が出にくくなる
  • 既存の自社CTAで一番反応が良かったものを1つ貼って「このトーンを基準にして」と伝えると、ブレが減る

8. コンテンツカレンダー骨子

用途

1ヶ月〜3ヶ月のコンテンツ計画の、最初の骨組みを作る。テーマの偏りチェックにも使える。

そのまま貼れるプロンプト

以下の条件で、{期間}分のコンテンツカレンダー骨子を作ってください。

- 媒体: {媒体（ブログ / SNS / メルマガ など。複数可）}
- 主な発信目的: {目的（信頼形成 / 認知拡大 / リード獲得 など）}
- 想定読者: {読者像}
- 既に出してきたテーマ: {直近で出したテーマを箇条書きで}
- 避けたいテーマ: {あれば}
- 投稿頻度: {頻度（例: 週3本 × 4週間）}

出力形式:
週ごとに以下を埋める
- 各回のテーマ案
- そのテーマで扱う切り口
- 媒体（該当する場合）
- 過去テーマと被りそうな点（あれば指摘）
- このテーマで読者に何を持ち帰ってほしいか

最後に、全体を通したテーマの偏り（似た領域に集中していないか）を一言コメントしてください。

入れ替える変数

  • {期間}: 1ヶ月 / 4週間 / 12週間 など
  • {媒体}: 単一でも複数でも
  • {頻度}: 現実的な本数で

使うときの注意

  • AIは「キレイに並んだ計画」を出す傾向がある。実務ではイレギュラーが入るので、3割は空き枠として残す
  • 出てきたテーマをそのまま採用すると、全体的にどこかで見たような構成になりやすい。1〜2本は社内独自のテーマに差し替える
  • 過去テーマを貼り忘れると、似たテーマが繰り返される

出力を良くする追加指示

  • 「同じツール名・同じサービス名は週内に2回以上登場させないでください」と縛ると、被りが減る
  • 「初心者向け / 中級者向け / 上級者向け」の比率を指定すると、レベル感のバランスが整う

9. 簡易SEO骨子

用途

記事作成前の、SEO観点での最初の骨組み。本格的なキーワード調査の前段階に使う。

そのまま貼れるプロンプト

以下のテーマで、SEO記事の骨子を1本分作ってください。

- メインキーワード: {メインKW}
- 関連キーワード（あれば）: {関連KW}
- 想定読者: {読者像}
- 検索意図の仮説: {読者が何を知りたくて検索しているかの仮説}
- 想定文字数: {文字数}

出力形式:
1. タイトル候補3つ（各32字以内）
2. リード文（200字程度）
3. 見出し構成（H2、H3まで。10〜15個程度）
4. 各見出しで触れるべきポイントを1〜2行
5. 検索意図とズレそうな見出しがあれば指摘
6. 内部リンクで紐づけたい関連テーマの候補
7. 注意点として、推測で書いている部分とデータで裏付けた方がよい部分を分けてください

避けたい表現: 「絶対」「必ず」「最強」「完全網羅」のような煽り。

入れ替える変数

  • {メインKW / 関連KW}: 検索キーワード
  • {読者像}: 想定読者
  • {検索意図の仮説}: 「○○を初めて使う人が、まず何を知りたいか」のような形
  • {文字数}: 2,000〜3,000字 など

使うときの注意

  • これは「骨子のたたき」。本格的にやるなら、検索ボリューム・競合上位記事・サジェスト調査などを別途行う
  • AIは「網羅型の見出し」を作りがち。全部書こうとすると冗長になるので、最終的に削る
  • 検索意図の仮説が間違っていると、骨子全体がズレる。仮説は社内で1度確認する

出力を良くする追加指示

  • 「検索上位の構成を真似るのではなく、検索意図に答える独自の流れにしてください」と入れると、テンプレ化を防げる
  • 「初心者がつまずきやすい部分を、見出しレベルで明示してください」と加えると、読者目線の骨子になる

10. 読者ペルソナ言い換え

用途

できあがった原稿を、別の読者層に向けて言い換える。同じ内容を媒体ごとに出し分ける時にも使える。

そのまま貼れるプロンプト

以下の原稿を、読者層を変えて書き直してください。

元の原稿:
{原稿本文}

元の想定読者:
{元の読者像}

新しい想定読者:
{新しい読者像}

書き直す時の条件:
- 内容の本質（伝えたい結論・根拠）は変えない
- 専門用語の使い方を新しい読者層に合わせる
- 例え・たとえ話を新しい読者の日常に近いものに差し替える
- 文体のトーン（敬体・常体）は新しい読者に合わせて選ぶ
- 元の原稿で「効きそうだった部分」をなるべく残す
- 文字数は元の原稿の±20%以内に収める

出力形式:
1. 書き直した原稿
2. 主な変更点を3〜5個（どこをどう変えたか）
3. この読者層には合わない可能性のある残し部分（あれば）

入れ替える変数

  • {原稿本文}: 既存の原稿
  • {元の読者像 / 新しい読者像}: 具体的に

使うときの注意

  • 言い換えのつもりが「中身が薄まる」ことがある。元の論点が残っているか必ず確認する
  • AIは新しい読者向けに「やさしくしすぎる」傾向がある。読者がプロ寄りなら、過度な平易化は逆効果になる
  • ブランドのトーンが媒体ごとに違うなら、トーンガイドも一緒に渡す

出力を良くする追加指示

  • 「専門用語は完全に置き換えるのではなく、必要な用語は残し、補足を一文で添える形にしてください」と指示すると、内容の厚みが残る
  • 元の原稿で削られたくない一文があれば「この一文は必ず残してください: 〜」と明示する

使い方の注意（全テンプレ共通）

ここまで紹介した10個のテンプレは、どれも「最初のたたき」「考える前の整理」を任せるためのものです。

そのまま完成原稿として使うのではなく、人間が判断と仕上げを担当することを前提にしています。

具体的には、次の3つを意識すると失敗しにくくなります。

1つ目は、個人情報・機密情報は必ず匿名化してから貼ることです。お客様の声や社内データを使う時、特に気をつけてください。

2つ目は、AIの出力は「ありそうな答え」になりやすいということです。煽り表現・断定・誇張が混ざりやすいので、最終チェックで弱める癖をつけると安全です。

3つ目は、同じテンプレを使い続けると出力が均質化することです。月に1回はテンプレ自体を見直すか、追加指示で切り口を変えると、表現が固まりません。

最後に

10個のテンプレを並べましたが、最初から全部使う必要はありません。

業務の中で「ここに毎回時間を取られているな」と感じる場面が1つあれば、そこに当てはまるテンプレから試してみるのが一番自然な始め方です。

使ってみて違和感のあった部分は、自分の業務に合わせてプロンプトを書き換えてください。テンプレは固定のものではなく、現場で育てていくものです。
